# Chorus privacy policy

Last updated: June 22, 2026

Chorus is a local AI chat application. It does not require a Chorus account or a Chorus chat server.

## Data stored on the device

Chorus stores chat history, drafts, settings, attachments, model choices, and saved memories in the application's local container. This data stays on the device unless the user sends a model request, shares a conversation, creates a device backup, or exports a file.

## Model requests

When the user sends a chat message, Chorus sends the request and the conversation context needed to answer it to the model provider selected by the user. The iPhone chat flow uses OpenRouter. The provider may process or retain data under its own terms and privacy policy.

## Memory

Memory is disabled by default. If the user enables Memory, Chorus may send selected user text to OpenAI to extract durable facts and create search embeddings. Chorus stores the resulting memories and embeddings in the local database. The user can inspect and delete memories inside the app.

Automatic learning is a separate setting. If it is enabled, Chorus may review user-authored messages after a conversation becomes idle. Chorus does not use assistant messages, web results, or attachment contents as the source of an automatic memory.

## API keys

The user provides their own provider API keys. Chorus stores those keys in its application data container and uses them only to connect directly to the selected provider.

## Analytics

The iPhone build does not use chat content for advertising or cross-app tracking. If diagnostics or analytics are enabled in a future release, this policy and the App Store privacy details must be updated before that release.

## Sharing and exports

The user can export or share a conversation as Markdown and JSON files. Chorus only shares those files after the user chooses a destination in the iOS share sheet.

## Deletion

The user can delete chats and memories inside the app. The user can remove all saved memories from Settings. Deleting the app removes its local application container from the device, subject to any device backup controlled by the user.
